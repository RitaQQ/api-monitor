# 測試案例管理系統

## 功能概述

這是一個完整的測試案例管理系統，採用 User Story 格式撰寫測試案例，支援專案化測試執行和詳細的統計報告。

## 主要功能

### 1. 測試案例管理 (Test Case Management)

#### 功能特色：
- **User Story 格式**：每個測試案例包含使用者角色、功能描述、驗收條件
- **Excel 風格編輯**：支援在表格中直接編輯，即時儲存
- **產品標籤管理**：可建立和管理產品標籤，一個測項可關聯多個產品
- **批量操作**：支援多選刪除、複製貼上
- **即時搜尋篩選**：可按照各欄位進行篩選和搜尋

#### 測試案例欄位：
- **ID**：自動生成的唯一識別碼
- **標題**：測試案例的簡短描述
- **使用者角色**：如「作為一位用戶」
- **功能描述**：如「我希望能...」
- **驗收條件**：可列多個條列的測試標準
- **測試備註**：額外的測試說明
- **產品標籤**：關聯的產品分類

### 2. 測試專案管理 (Test Projects)

#### 專案建立流程：
1. **基本資訊**：設定專案名稱、測試日期、負責人
2. **選擇測試案例**：從現有測項中選擇要測試的案例
3. **確認建立**：檢視所選項目並建立專案

#### 測試執行：
- **即時記錄結果**：通過/失敗/待測試
- **詳細備註**：記錄測試過程和發現的問題
- **已知問題追蹤**：記錄和管理已知的問題
- **進度追蹤**：即時顯示測試完成度

### 3. 統計報表功能

#### 統計指標：
- **整體通過率**：所有測項的通過百分比
- **產品別統計**：按產品標籤分組的通過率
- **風險評估**：根據失敗率自動評估風險等級
- **完成度追蹤**：測試進度的實時監控

#### 分析功能：
- **趨勢分析**：比較多個專案的表現
- **問題識別**：自動標示表現不佳的產品
- **建議生成**：根據測試結果提供改善建議

### 4. PDF 報告匯出

#### 報告內容：
- **執行摘要**：專案概況和關鍵指標
- **詳細統計**：完整的數據分析和圖表
- **產品分析**：各產品的表現分析
- **測試案例詳情**：所有測項的執行結果
- **建議與結論**：基於數據的具體建議

## 使用指南

### 首次使用

1. **安裝依賴**：
   ```bash
   pip install -r requirements.txt
   ```

2. **啟動系統**：
   ```bash
   source venv/bin/activate  # macOS
   python simple_app.py
   ```

3. **訪問介面**：
   - 主頁：http://localhost:5001
   - 測試案例管理：http://localhost:5001/test-case-management
   - 測試專案：http://localhost:5001/test-projects

### 建立第一個測試案例

1. 點擊導航選單中的「測試案例管理」
2. 點擊「新增測項」按鈕
3. 直接在表格中編輯：
   - 標題：簡短描述測試內容
   - 使用者角色：如「作為一位註冊用戶」
   - 功能描述：如「我希望能夠登入系統」
   - 點擊「編輯驗收條件」按鈕新增具體的驗收標準
4. 設定產品標籤（可選）
5. 系統會自動儲存

### 執行測試專案

1. 點擊導航選單中的「測試專案」
2. 點擊「建立新專案」
3. 依照精靈步驟：
   - 填寫專案基本資訊
   - 選擇要測試的案例（可搜尋和篩選）
   - 確認並建立專案
4. 在專案詳情中記錄測試結果：
   - 點擊「通過」或「失敗」按鈕
   - 填寫測試備註和已知問題
   - 系統即時更新統計數據

### 產生測試報告

1. 在專案詳情頁面點擊「匯出PDF」
2. 系統會生成包含以下內容的報告：
   - 專案概況和關鍵指標
   - 詳細的統計圖表
   - 各產品的表現分析
   - 測試案例執行詳情
   - 基於數據的改善建議

## 系統架構

### 資料模型
- **TestCase**：測試案例模型
- **ProductTag**：產品標籤模型
- **TestProject**：測試專案模型
- **TestResult**：測試結果模型

### 核心組件
- **TestCaseManager**：測試案例管理器
- **ReportGenerator**：報告生成器
- **PDFExporter**：PDF匯出器

### 資料儲存
- 使用 JSON 檔案儲存：
  - `data/test_cases.json`：測試案例資料
  - `data/product_tags.json`：產品標籤資料
  - `data/test_projects.json`：測試專案資料

## API 文檔

### 測試案例 API
- `GET /api/test-cases`：取得所有測試案例
- `POST /api/test-cases`：建立新測試案例
- `PUT /api/test-cases/{id}`：更新測試案例
- `DELETE /api/test-cases/{id}`：刪除測試案例

### 專案管理 API
- `GET /api/test-projects`：取得所有測試專案
- `POST /api/test-projects`：建立新測試專案
- `PUT /api/test-projects/{id}/results`：更新測試結果

### 報告 API
- `GET /api/test-projects/{id}/statistics`：取得專案統計
- `POST /api/test-projects/{id}/export-pdf`：匯出PDF報告

## 最佳實踐

### 撰寫測試案例
1. **使用 User Story 格式**：
   - 使用者角色要具體明確
   - 功能描述要清晰易懂
   - 驗收條件要可測試和可驗證

2. **合理使用產品標籤**：
   - 按功能模組分類
   - 避免標籤過於細分
   - 保持標籤命名一致性

### 專案管理
1. **專案命名**：包含版本號或測試範圍
2. **測試案例選擇**：根據風險和重要性選擇
3. **結果記錄**：及時記錄測試過程和發現

### 報告分析
1. **定期生成報告**：追蹤品質趨勢
2. **關注風險指標**：重點改善高風險產品
3. **數據驅動決策**：基於統計數據制定策略

## 故障排除

### 常見問題

1. **PDF 匯出失敗**
   - 檢查是否安裝了 reportlab 和 matplotlib
   - 確認有足夠的磁碟空間

2. **測試案例無法儲存**
   - 檢查 data 目錄的寫入權限
   - 確認 JSON 檔案格式正確

3. **介面載入緩慢**
   - 檢查測試案例數量是否過多
   - 考慮使用篩選功能減少顯示數量
